162            /**
163             * Returns an input stream for this socket.
164             * 
165             * <p>
166             * If this socket has an associated channel then the resulting input stream
167             * delegates all of its operations to the channel. If the channel is in
168             * non-blocking mode then the input stream's <tt>read</tt> operations will
169             * throw an {@link java.nio.channels.IllegalBlockingModeException}.
170             * 
171             * <p>
172             * Under abnormal conditions the underlying connection may be broken by the
173             * remote host or the network software (for example a connection reset in
174             * the case of TCP connections). When a broken connection is detected by the
175             * network software the following applies to the returned input stream :-
176             * 
177             * <ul>
178             * 
179             * <li>
180             * <p>
181             * The network software may discard bytes that are buffered by the socket.
182             * Bytes that aren't discarded by the network software can be read using
183             * {@link java.io.InputStream#read read}.
184             * 
185             * <li>
186             * <p>
187             * If there are no bytes buffered on the socket, or all buffered bytes have
188             * been consumed by {@link java.io.InputStream#read read}, then all
189             * subsequent calls to {@link java.io.InputStream#read read} will throw an
190             * {@link java.io.IOException IOException}.
191             * 
192             * <li>
193             * <p>
194             * If there are no bytes buffered on the socket, and the socket has not been
195             * closed using {@link #close close}, then
196             * {@link java.io.InputStream#available available} will return
197             * <code>0</code>.
198             * 
199             * </ul>
200             * 
201             * <p>
202             * Closing the returned {@link java.io.InputStream InputStream} will close
203             * the associated socket.
204             * 
205             * @return an input stream for reading bytes from this socket.
206             * @exception IOException
207             *                if an I/O error occurs when creating the input stream, the
208             *                socket is closed, the socket is not connected, or the
209             *                socket input has been shutdown using
210             *                {@link #shutdownInput()}
211             * 
212             *                revised 1.4 spec JSR-51
213             */

428            /**
429             * Sets the SO_RCVBUF option to the specified value for this <tt>Socket</tt>
430             * . The SO_RCVBUF option is used by the platform's networking code as a
431             * hint for the size to set the underlying network I/O buffers.
432             * 
433             * <p>
434             * Increasing the receive buffer size can increase the performance of
435             * network I/O for high-volume connection, while decreasing it can help
436             * reduce the backlog of incoming data.
437             * 
438             * <p>
439             * Because SO_RCVBUF is a hint, applications that want to verify what size
440             * the buffers were set to should call {@link #getReceiveBufferSize()}.
441             * 
442             * <p>
443             * The value of SO_RCVBUF is also used to set the TCP receive window that is
444             * advertized to the remote peer. Generally, the window size can be modified
445             * at any time when a socket is connected. However, if a receive window
446             * larger than 64K is required then this must be requested <B>before</B> the
447             * socket is connected to the remote peer. There are two cases to be aware
448             * of:
449             * <p>
450             * <ol>
451             * <li>For sockets accepted from a ServerSocket, this must be done by
452             * calling {@link ServerSocket#setReceiveBufferSize(int)} before the
453             * ServerSocket is bound to a local address.
454             * <p></li>
455             * <li>For client sockets, setReceiveBufferSize() must be called before
456             * connecting the socket to its remote peer.
457             * <p></li>
458             * </ol>
459             * 
460             * @param size
461             *            the size to which to set the receive buffer size. This value
462             *            must be greater than 0.
463             * 
464             * @exception IllegalArgumentException
465             *                if the value is 0 or is negative.
466             * 
467             * @exception SocketException
468             *                if there is an error in the underlying protocol, such as a
469             *                TCP error.
470             * 
471             * @see #getReceiveBufferSize()
472             * @see ServerSocket#setReceiveBufferSize(int)
473             * @since 1.2
474             */
475            void setReceiveBufferSize(int size) throws SocketException;

482             * @exception SocketException
483             *                if there is an error in the underlying protocol, such as a

516            /**
517             * Sets traffic class or type-of-service octet in the IP header for packets
518             * sent from this Socket. As the underlying network implementation may
519             * ignore this value applications should consider it a hint.
520             * 
521             * <P>
522             * The tc <B>must</B> be in the range <code> 0 <= tc <=
523             * 255</code> or an IllegalArgumentException will be thrown.
524             * <p>
525             * Notes:
526             * <p>
527             * For Internet Protocol v4 the value consists of an octet with precedence
528             * and TOS fields as detailed in RFC 1349. The TOS field is bitset created
529             * by bitwise-or'ing values such the following :-
530             * <p>
531             * <UL>
532             * <LI><CODE>IPTOS_LOWCOST (0x02)</CODE></LI>
533             * <LI><CODE>IPTOS_RELIABILITY (0x04)</CODE></LI>
534             * <LI><CODE>IPTOS_THROUGHPUT (0x08)</CODE></LI>
535             * <LI><CODE>IPTOS_LOWDELAY (0x10)</CODE></LI>
536             * </UL>
537             * The last low order bit is always ignored as this corresponds to the MBZ
538             * (must be zero) bit.
539             * <p>
540             * Setting bits in the precedence field may result in a SocketException
541             * indicating that the operation is not permitted.
542             * <p>
543             * As RFC 1122 section 4.2.4.2 indicates, a compliant TCP implementation
544             * should, but is not required to, let application change the TOS field
545             * during the lifetime of a connection. So whether the type-of-service field
546             * can be changed after the TCP connection has been established depends on
547             * the implementation in the underlying platform. Applications should not
548             * assume that they can change the TOS field after the connection.
549             * <p>
550             * For Internet Protocol v6 <code>tc</code> is the value that would be
551             * placed into the sin6_flowinfo field of the IP header.
552             * 
553             * @param tc
554             *            an <code>int</code> value for the bitset.
555             * @throws SocketException
556             *             if there is an error setting the traffic class or
557             *             type-of-service
558             * @since 1.4
559             * @see #getTrafficClass
560             */
561            void setTrafficClass(int tc) throws SocketException;
562    
563            /**
564             * Gets traffic class or type-of-service in the IP header for packets sent
565             * from this Socket
566             * <p>
567             * As the underlying network implementation may ignore the traffic class or
568             * type-of-service set using {@link #setTrafficClass(int)} this method may
569             * return a different value than was previously set using the
570             * {@link #setTrafficClass(int)} method on this Socket.
571             * 
572             * @return the traffic class or type-of-service already set
573             * @throws SocketException
574             *             if there is an error obtaining the traffic class or
575             *             type-of-service value.
576             * @since 1.4
577             * @see #setTrafficClass(int)
578             */
579            int getTrafficClass() throws SocketException;

581            /**
582             * Enable/disable the SO_REUSEADDR socket option.
583             * <p>
584             * When a TCP connection is closed the connection may remain in a timeout
585             * state for a period of time after the connection is closed (typically
586             * known as the <tt>TIME_WAIT</tt> state or <tt>2MSL</tt> wait state). For
587             * applications using a well known socket address or port it may not be
588             * possible to bind a socket to the required <tt>SocketAddress</tt> if there
589             * is a connection in the timeout state involving the socket address or
590             * port.
591             * <p>
592             * Enabling <tt>SO_REUSEADDR</tt> prior to binding the socket using
593             * {@link #bind(SocketAddress)} allows the socket to be bound even though a
594             * previous connection is in a timeout state.
595             * <p>
596             * When a <tt>Socket</tt> is created the initial setting of
597             * <tt>SO_REUSEADDR</tt> is disabled.
598             * <p>
599             * The behaviour when <tt>SO_REUSEADDR</tt> is enabled or disabled after a
600             * socket is bound (See {@link #isBound()}) is not defined.
601             * 
602             * @param on
603             *            whether to enable or disable the socket option
604             * @exception SocketException
605             *                if an error occurs enabling or disabling the
606             *                <tt>SO_RESUEADDR</tt> socket option, or the socket is
607             *                closed.
608             * @since 1.4
609             * @see #getReuseAddress()
610             * @see #bind(SocketAddress)
611             * @see #isClosed()
612             * @see #isBound()
613             */
614            void setReuseAddress(boolean on) throws SocketException;

624             * @since 1.4

642             * OutputStream}.
643             * 
644             * <p>
645             * If this socket has an associated channel then the channel is closed as

647             *

653            void close() throws IOException;

672            void shutdownInput() throws IOException;

744            /**
745             * Sets performance preferences for this socket.
746             * 
747             * <p>
748             * Sockets use the TCP/IP protocol by default. Some implementations may
749             * offer alternative protocols which have different performance
750             * characteristics than TCP/IP. This method allows the application to
751             * express its own preferences as to how these tradeoffs should be made when
752             * the implementation chooses from the available protocols.
753             * 
754             * <p>
755             * Performance preferences are described by three integers whose values
756             * indicate the relative importance of short connection time, low latency,
757             * and high bandwidth. The absolute values of the integers are irrelevant;
758             * in order to choose a protocol the values are simply compared, with larger
759             * values indicating stronger preferences. Negative values represent a lower
760             * priority than positive values. If the application prefers short
761             * connection time over both low latency and high bandwidth, for example,
762             * then it could invoke this method with the values <tt>(1, 0, 0)</tt>. If
763             * the application prefers high bandwidth above low latency, and low latency
764             * above short connection time, then it could invoke this method with the
765             * values <tt>(0, 1, 2)</tt>.
766             * 
767             * <p>
768             * Invoking this method after this socket has been connected will have no
769             * effect.
770             * 
771             * @param connectionTime
772             *            An <tt>int</tt> expressing the relative importance of a short
773             *            connection time
774             * 
775             * @param latency
776             *            An <tt>int</tt> expressing the relative importance of low
777             *            latency
778             * 
779             * @param bandwidth
780             *            An <tt>int</tt> expressing the relative importance of high
781             *            bandwidth
782             * 
783             * @since 1.5
784             */
785            void setPerformancePreferences(int connectionTime, int latency,
786                            int bandwidth);
